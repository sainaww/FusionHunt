These were sample rows that were downloaded from UCSC Table browser (hg18).

Detailed information about the annotation file formats can be found at http://genome.ucsc.edu/FAQ/FAQformat.html and http://mblab.wustl.edu/GTF2.html.

Where:
#bin:         Bin value (not used by the pipeline)
name:         Transcript ID
chrom:        Chromosome
strand:       Genomic strand of the transcript
txSt: 	      Genomic position of the transcription start site.
txEnd:        Genomic position of the transcription end site.
cdsStart:     Genomic position of the ORF (or protein coding sequence) start.
cdsEnd:       End of coding sequence.
exonCount:    Number of exons in the transcript.
exonStarts:   Comma separated list of genomic psositions of exon starts. The number of items in this list should correspond to exonCount.
exonEnds:     Comma separated list of exon end positions. The number of items in this list should correspond to exonCount.

Note: Rest of the fields will be ignored (if there are any)

Where:
chrom - The name of the chromosome (e.g. chr3, chrY, chr2_random).
chromStart - The starting position of the feature in the chromosome.
chromEnd - The ending position of the feature in the chromosome.
name - Transcript ID.
score - A score between 0 and 1000 (not used by the pipeline).
strand - Genomic strand of the transcript.
thickStart - The starting position at which the feature is drawn thickly (for example, the start codon in gene displays).
thickEnd - The ending position at which the feature is drawn thickly (for example, the stop codon in gene displays).
itemRgb - An RGB value of the form R,G,B (not used by the pipeline).
blockCount - The number of blocks (exons).
blockSizes - A comma-separated list of the block sizes (exon sizes). The number of items in this list should correspond to blockCount.
blockStarts - A comma-separated list of block starts. All of the blockStart positions should be calculated relative to chromStart. The number of items in this list should correspond to blockCount.

Note: Rest of the fields will be ignored (if there are any)
